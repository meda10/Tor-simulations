Recommendations

  • Path simulation
    • While using small simulation use five nodes of each at most. Otherwise the graph would lose its structure
    • While using large simulation use anywhere from 25 to 250 nodes
    • Number of simulations should not be higher than 120 or the graph size will be too large (up to 40MB) and browser will have problem loading it
    • For small simulation path selection is always random
  • Onion service simulation
    • Maximum size should be around 250 nodes
  • Correlation attack
    • Simulator generates node bandwidth between 1 KB/s and 2500 KB/s (adjust your values accordingly)
    • Number of simulations can be set up to 10 000
    • Use at least 30 nodes for your simulation
  • Exit Attack: Eavesdropping on communication
    • Simulator generates node bandwidth between 1 KB/s and 2500 KB/s (adjust your values accordingly)
    • Number of simulations can be set up to 10 000
    • Use at least 30 nodes for your simulation
  • Multiple simulations
    • Recommended number of simulations is between six and twelve
    • Parameter Number of simulations should be around 2 000
    • If you input too many simulations it will take too long and sometimes your browser may break the connection

Parameters

General parameters
  • Simulation type - chooses the type of simulations that will be executed
  • Guard - number of friendly nodes with flag guard
  • Exit - number of friendly nodes with flag exit
  • Middle - number of friendly nodes without flag guard or exit
  • Guard & exit - number of friendly nodes with flag guard and exit
  • Remove duplicate paths - removes paths that already exist
  • Same bandwidth - gives every node same bandwidth (default 500 KB/s)
  • Bandwidth guard (optional) KB/s - gives every guard same bandwidth
  • Bandwidth middle (optional) KB/s - gives every middle same bandwidth
  • Bandwidth exit (optional) KB/s - gives every exit same bandwidth
  • Number of simulations - this is the number of circuits that will be generated
  • Simulation size - selects the size of the graph that will be created (large/small- only for Path simulations)
  • Path selection - chooses which type will be used (random/1 guard/3guard - only for Path simulations)
  • Nodes - number of nodes (splits 50/50 between guard and exit)
  • Encryption(%) - percentage of end-to-end encrypted traffic
  • ID occurrence(%) - percentage of the chance that in given circuit will appear some ID
  • Adversary guard - number of adversary guards
  • Adversary exit - number of adversary exits
  • Adversary guard bandwidth (KB/s) - bandwidth of adversary guards in KB/s
  • Adversary exit bandwidth (KB/s) - bandwidth of adversary exits in KB/s
Custom nodes
  • Name - name of the custom node
  • IP - IP address for a custom node
  • Bandwidth (KB/s) - bandwidth in KB/s for a custom node
Multiple simulations
  • F-guard - number of friendly nodes with flag guard
  • F-exit - number of friendly nodes with flag exit
  • F-guard bandwidth - bandwidth of friendly guards
  • F-exit bandwidth - bandwidth of friendly exits
  • ADV-guard - number of adversary guards
  • ADV-exit - number of adversary exits
  • A-guard bandwidth - bandwidth of adversary guards in KB/s
  • A-exit bandwidth - bandwidth of adversary exits in KB/s
  • Encryption(%) - percentage of end-to-end encrypted traffic
  • ID occurrence(%) - percentage of the chance that in given circuit will appear some ID
