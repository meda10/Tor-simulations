Path simulation

This simulation shows selection of path in Tor network, thanks to parameter path selection we can see which approach was used in the past by the Tor network.

Graph shows circuits in Tor network and differentiates nodes according to their position in circuit. Graph is interactive and allows users to switch between individual paths. There are also tables, first of which shows circuits and their assigned number in the graph. Second table shows individual nodes, their IP address, bandwidth and number of times a node was used in circuit.

Onion service simulation

This simulation explains step by step how can user connect to onion service. It shows communication between onion service, introduction points and randezvous point.

Graph shows Tor network, for better understanding are important relays marked by pictures. Final graph is interactive and user can see every step individually.

Correlation attack simulation

This simulations demonstrates how the correlation attack works. For a successful attack, attacker needs to be in control of both guard and exit nodes.

Graph shows circuits in which attacker controlled both guard and exit, graph is interactive and enables user to see paths individually. Color of the nodes is saturated according to how many times was the node used in circuit. Nodes are also distinct by color, enemy nodes are red and friendly nodes are blue. There are also tables, first of which shows circuits and their assigned number in the graph. Second table shows individual nodes, their IP address, bandwidth, number of times a node was used in circuit and how many end-to-end encrypted communication passed through the node. Further statistics can be found in tab 'statistics'.

Simulation of eavesdropping on exit nodes

This simulation concentrates on attacker that controls exit relay and eavesdrops on unencrypted communication

Thanks to parameter ID occurrence we can model chances of circuit containing ID that can be stolen. Color of the nodes is saturated according to how many IDs have passed through the given node. Nodes are also distinct by color, enemy nodes are red and friendly nodes are blue. There are also tables, first of which shows circuits and their assigned number in the graph. Second table shows individual nodes, their bandwidth, how many IDs have passed through the given node and how many IDs were stolen, both in numbers and percents (when attacker steals ID from exit node, the ID shows as stolen for every node in circuit). Further statistics can be found in tab 'statistics'.

Multiple simulation

This allows to run multiple simulations consecutively, each of them can have their own parameters that you can change and watch how it affects outputs. Final graphs and statistics compare results of all simulations.
